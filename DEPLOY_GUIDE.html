☁️ Cloudflare Deployment Guide

Step-by-step: Deploy FitGirl Helper as a website using Cloudflare Workers + Pages + GitHub Actions

☁️

Overview — What You're Building

3 pieces that work together. Total setup time: ~10 minutes.

architecture
🌐 Cloudflare Pages — hosts the website (what users see)
↕ calls API
⚡ Cloudflare Worker — the hidden backend (scrapes pages, triggers extraction)
↕ triggers workflow
🤖 GitHub Actions — runs Chrome to bypass Cloudflare & extract direct links
📁
Files you'll need:
📄 WORKER_CODE.js → paste into Cloudflare Worker
📁 site/index.html → deployed via Cloudflare Pages
📁 .github/workflows/ + 📁 scripts/ → pushed to GitHub repo
▼
1

Push Code to GitHub one-time ~2 min

Create a GitHub repo with the workflow + extraction script.

  1. Go to github.com/new and create a new repository (e.g. fitgirl_helper_web)
  2. Clone it locally and copy these files into it:
    # Your repo should look like this: fitgirl_helper_web/ ├── .github/workflows/ │ └── extract-links.yml # GitHub Actions workflow ├── scripts/ │ └── extract.py # Python extraction script ├── site/ │ └── index.html # Website frontend └── WORKER_CODE.js # Copy-paste into Cloudflare
  3. Push to GitHub:
    git add -A git commit -m "initial commit" git push origin main
  4. Create the gh-pages branch (where extraction results go):
    git checkout --orphan gh-pages git rm -rf . mkdir results touch results/.gitkeep git add results/ git commit -m "init gh-pages" git push origin gh-pages git checkout main
✅
Your repo is ready. Note the repo name — you'll need it in Step 3.
▼
2

Create GitHub Token one-time ~1 min

The Worker needs a token to trigger GitHub Actions on your behalf.

  1. Go to github.com/settings/tokens
  2. Click "Generate new token (classic)"
  3. Name it: fitgirl-worker
  4. Set expiration to No expiration (or your preference)
  5. Check these scopes:

    ✅ repo (full control of private repositories)
    ✅ workflow (update GitHub Action workflows)
  6. Click "Generate token"
  7. ⚠️
    COPY THE TOKEN NOW! It starts with ghp_ and you won't see it again. Save it somewhere safe temporarily.
▼
3

Create the Cloudflare Worker one-time ~3 min

This is the hidden backend that scrapes pages and triggers extraction.

3a. Create the Worker

  1. Go to Cloudflare Dashboard → Workers & Pages
  2. Click the blue "Create" button
  3. You'll see this screen:

    Select a method
    Ship something new
    Select from a git repo, drag and drop your code or select a template.
    Continue with GitHub
    Connect GitLab
    👉 Start with Hello World!
    Select a template
  4. Click "Start with Hello World!"
  5. Set the Worker name to: fitgirl-helper-api
    Name your Worker
    Worker name
    fitgirl-helper-api
    Your Worker will be deployed to: fitgirl-helper-api.your-subdomain.workers.dev
    Deploy
  6. Click "Deploy" — it creates a hello-world Worker
  7. You'll see a success screen. Click "Edit Code"
    Worker deployed successfully!
    🎉
    Your Worker is live!
    👉 Edit Code
    Configure Worker
  8. Select ALL the existing code in the editor (Ctrl+A) and DELETE it

  10. Copy ALL of WORKER_CODE.js and paste it into the Cloudflare editor
  11. Click "Deploy" (top-right blue button)
✅
Worker code is deployed! Now add secrets.

3b. Add Secrets to the Worker

  1. Go back to the Worker overview (click the Worker name in the sidebar)
  2. Click "Settings" tab at the top
  3. In the left sidebar, click "Variables and Secrets"

    Environment Variables
    + Add
  4. Click "+ Add" and add these three variables:
Variable NameTypeValue
GITHUB_TOKEN 🔒 Secret The ghp_xxx token you created in Step 2
GITHUB_OWNER 📝 Text Your GitHub username (e.g. MacReyhan)
GITHUB_REPO 📝 Text Your repo name (e.g. fitgirl_helper_web)

▼
4

Set Worker URL in Website ~30 sec

Tell the frontend where its backend lives.

  1. Open 📄 site/index.html
  2. Find this line near the bottom (in the <script> section):
    const API_URL = '%%WORKER_URL%%';
  3. Replace %%WORKER_URL%% with your actual Worker URL:
    const API_URL = 'https://fitgirl-helper-api.your-subdomain.workers.dev';
  4. Save the file and commit + push to GitHub:
    git add site/index.html git commit -m "set worker URL" git push origin main
▼
5

Deploy Website to Cloudflare Pages one-time ~3 min

Host the frontend for free on Cloudflare Pages.

  1. Go to Cloudflare Dashboard → Workers & Pages
  2. Click the blue "Create" button
  3. This time, click "Continue with GitHub" (or "Pages" tab at the top)
    Pages → Connect to Git
    Connect your GitHub account
    Connect GitHub
  4. Authorize Cloudflare to access your GitHub (if not already done)
  5. Select your repository: fitgirl_helper_web
  6. Configure the build settings:
    Configure build settings
    Project name
    fitgirl-helper
    Production branch
    main
    Framework preset
    None
    Build command
    (leave empty)
    Build output directory 👈 IMPORTANT
    site
    Save and Deploy
  7. ⚠️
    Make sure Build output directory is set to site (not / or public). This tells Cloudflare to only serve the site/ folder.
  8. Click "Save and Deploy"
  9. Wait ~30 seconds for deployment
🎉
Your site is live!
URL: https://fitgirl-helper.pages.dev (or whatever you named it)
Every push to main will auto-redeploy.

Here's what happens when someone uses your site:

User experience
1️⃣ User opens fitgirl-helper.pages.dev
2️⃣ Pastes a FitGirl URL → clicks Fetch
3️⃣ → Website calls your Worker → Worker scrapes the page → returns links instantly
4️⃣ User selects parts → clicks Extract
5️⃣ → Worker triggers GitHub Actions → Chrome runs → bypasses Cloudflare → saves results
6️⃣ → Website polls Worker → Worker checks GitHub → returns results when ready
7️⃣ Direct download links appear → user copies them! 🎯
The user never sees:
  • No GitHub tokens
  • No API configuration
  • No setup screens
  • No source code secrets

Troubleshooting

ProblemFix
"Server not configured"You forgot to add GITHUB_TOKEN / GITHUB_OWNER / GITHUB_REPO as Worker variables
Fetch works but Extract never finishesCheck your GitHub repo → Actions tab for failed runs. Common: wrong PAT scopes.
"GitHub 404" errorMake sure extract-links.yml exists in .github/workflows/ on the main branch
CORS error in browser consoleMake sure the Worker has the CORS headers (the provided code includes them)
Results never appear after extractionCheck that gh-pages branch exists. The workflow commits results there.
Want a custom domain?Cloudflare Pages → Custom domains → add your domain. Free SSL included.
